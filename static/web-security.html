安全なアプリケーションの作り方

安全なアプリケーションには適切な権限管理と脆弱性を生まないことが大切

脆弱性とは?

脆弱性とはセキュリティバグのことを指し、悪用できるバグとも言えます。 アプリケーションにバグはつきものですが、セキュリティーホールになりえるようなバグの例は以下になります。

  1. 個人情報が閲覧できる
  2. Webサイトの内容が書き換えられる
  3. サイトを閲覧した利用者のPCを感染させる
  4. コンピューター資源を勝手に使われる (暗号通貨のマイニング)
