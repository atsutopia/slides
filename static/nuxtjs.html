About Nuxt.js

About Nuxt.js

Nuxt.js は Vue アプリケーションを作成するフレームワークです. ユニバーサルアプリケーション,静的に生成されるアプリケーション,シングルページアプリケーションの中から作成するアプリケーションを選ぶことができます.

Nuxt.js採用の理由

  • SSR対応 - handle meta deta
  • ファイルベースRouter
  • コードの自動分割 - モダンな高速化テクニックのプリセット
    • refhttps://ja.nuxtjs.org/guide/

    JavaScriptフレームワーク比較

    Vue vs React vs Angular

    SPAフレームワークでも有名なReactやAngularと比較していく. 留意しておきたいのはフレームワークの取捨選択はあくまでケースバイケースとなる.

    with React

  • JSX vs Templates

    • Reactでは,すべてのコンポーネントは JSX を用いた 描画関数 (render) の中でそれらの UI を表現する. これはVueにおけるTemplatesのそれとは違い,独自のDSLが含まれる.

  • コミュニティ

    • Vue.jsではvuexやvue-routerなどのメジャーなライブラリは公式orgにおいて開発されている. 一方,Reactではreduxやreact-routerなどメジャーなライブラリもfacebookではなく独自に開発されている. fluxなどの公式に提供されているものが逆に流行っていないということもある. GitHub Star数では 138k > 129k でvueに軍配が上がる

    with Angular

  • Typescript

    • ほとんどすべてのドキュメントと学習リソースがTypeScriptベースのため,実質TypeScript を使用する必要がある.

  • コミュニティ

    • GitHub Star数では 138k > 48k でvueに軍配が上がる

    BFFフレームワーク比較

    About BFF (Backends For Frontends)

    BFFとは名前の通り,フロントエンドのためのバックエンド（サーバ）のことを指す. BFFのアーキテクチャパターンにより「フロントエンド専用のサーバを設けることでUIを構築しやすくする」効果と, 「フロントエンドとバックエンドの境界を設けてアーキテクチャのレイヤーを分割することで役割を明確にし、独立して開発を進めやすくする」効果が 期待できる.

    https://samnewman.io/patterns/architectural/bff/

    Nuxt.js vs Next.js

    Nuxt.js は Vueベースで作成されたフレームワークでNext.jsはReactベースで作成されたフレームワークである. どちらもSSRを実現するためにできていて,BFFが実現可能. プリセットの豊富さや周辺ライブラの豊富さはNuxt.jsに軍配があがる.
